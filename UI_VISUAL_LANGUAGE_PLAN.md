# UI Visual Language Plan

## Goal
Create a reusable first-party visual language for Theme Park Shark so the app stops feeling like patched utility UI and starts feeling like a premium game product.

## Phase 1: Reward System Visual Kit
- Resource icons
  - ride parts ✅
  - energy ✅
  - shelf progress
  - coin glow / upgrade tier
- Reward modal hierarchy
  - hero reward zone
  - secondary stat cards
  - shelf progress pill
  - premium CTA treatment
- Palette cleanup
  - teal/cyan = shelf / coin identity
  - gold = currency / celebration
  - violet = upgrade material
  - amber = energy
  - green only where XP truly needs it

## Phase 2: Progression UI Kit
- park shelf surfaces
- coin level / glow states
- progression chips
- mastery badges
- rarity frames

## Phase 3: Social / Identity UI Kit
- shark identity frames
- player prestige badges
- friend / team presence chips
- park-local presence markers

## Rules
- no emoji fallback for core game rewards
- every core resource gets first-party art
- no rainbow clutter without hierarchy
- hero reward should dominate reward moments
- support rewards should feel cohesive, not equal-weight chaos
